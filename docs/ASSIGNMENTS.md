# 추가 과제

기본 실습을 끝낸 뒤 학생들이 코드를 직접 바꿔보며 이해도를 확인할 수 있는 과제입니다.

## 과제 1. 카메라 해상도와 FPS 비교

목표: 해상도가 높아질수록 처리 속도와 화면 반응이 어떻게 달라지는지 확인한다.

수정할 파일:

```text
examples/06_opencv_camera_preview.py
```

진행:

1. 카메라 크기를 `320x240`, `640x480`, `1280x720`으로 바꿔가며 실행한다.
2. 각 해상도에서 화면이 얼마나 부드러운지 관찰한다.
3. 코드에 FPS 계산을 추가해 터미널에 출력한다.

제출:

- 실험한 해상도 3개
- 각 해상도에서 측정한 평균 FPS
- 가장 실습에 적합하다고 생각한 해상도와 이유

## 과제 2. 키보드로 사진 저장 기능 개선

목표: OpenCV 프리뷰 루프와 키 입력 처리를 이해한다.

수정할 파일:

```text
examples/07_opencv_capture_save.py
```

진행:

1. `s`를 누르면 사진이 저장되는 동작을 확인한다.
2. 저장 파일 이름에 날짜와 시간을 포함하도록 수정한다.
3. 저장할 때 화면 왼쪽 위에 `Saved` 문구가 1초 동안 보이게 한다.

제출:

- 수정한 코드
- 저장된 사진 2장
- 키 입력 처리 흐름 설명

## 과제 3. YOLO 탐지 결과 필터링

목표: YOLO 결과 객체 구조와 confidence 값을 이해한다.

수정할 파일:

```text
examples/10_yolo_ncnn_camera.py
```

진행:

1. confidence가 `0.5` 이상인 객체만 출력하도록 수정한다.
2. 특정 클래스만 출력하도록 수정한다. 예: `person`, `cup`, `cell phone`
3. 프레임마다 탐지된 객체 수를 화면에 표시한다.

제출:

- 필터링 기준
- 탐지 결과 화면 캡처
- confidence threshold를 높이면 결과가 어떻게 달라지는지 설명

## 과제 4. 간단한 실시간 응용 만들기

목표: 카메라 입력, OpenCV 처리, YOLO 결과를 하나의 응용으로 연결한다.

아래 중 하나를 선택한다.

- 사람이 감지되면 화면에 `PERSON DETECTED` 표시
- 특정 물체가 감지되면 사진 자동 저장
- 프레임 중앙에 십자선을 그리고 물체가 중앙에 들어오면 표시 색 변경
- 10초 동안 감지된 객체 이름과 횟수를 CSV 파일로 저장

제출:

- 실행 방법
- 수정한 코드
- 결과 이미지 또는 짧은 동영상
- 어떤 조건에서 오동작했는지 한 가지 이상 설명

## 권장 평가 기준

- 기본 동작을 실행할 수 있는가
- 코드에서 카메라 설정, 프레임 획득, 화면 출력 위치를 설명할 수 있는가
- OpenCV 키 입력과 저장 로직을 직접 수정할 수 있는가
- YOLO 결과에서 클래스명, confidence, bounding box를 읽어 활용할 수 있는가
- 실패 상황을 관찰하고 원인을 말할 수 있는가
